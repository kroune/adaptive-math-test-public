## 5+6. Переработка системы психологических вопросов (оба режима)

**Что сейчас:** в `useTestFlow.ts` мотивационные вопросы показываются: после выбора режима (перед первой темой) и после
завершения каждой темы. Вопрос выбирается случайно из общего пула. Нет различия по типу/моменту показа.

**Что нужно — 4 момента показа (одинаковые для обоих режимов):**

| Момент                          | Линейный                        | Адаптивный                               |
|---------------------------------|---------------------------------|------------------------------------------|
| Перед началом теста             | Общий вопрос                    | Общий вопрос                             |
| Перед каждой темой              | Показать название темы + вопрос | После выбора темы, перед первым заданием |
| После последнего задания в теме | Вопрос по итогу темы            | Вопрос по итогу темы                     |
| В конце всего теста             | Финальный вопрос                | Финальный вопрос                         |

**Ключевое уточнение:** каждый вопрос — уникальный. Соня сама назначает какой конкретный вопрос показывается в каком
месте. Это не случайный выбор из пула.

**Что делать:**

1. Добавить поле `timing` в таблицу `motivation_questions`:
   ```sql
   timing text NOT NULL DEFAULT 'before_test'
   -- значения: 'before_test', 'before_topic', 'after_topic', 'after_test'
   ```
2. Добавить поле `topic_id uuid REFERENCES topics` — для привязки вопроса к конкретной теме (для `before_topic` и
   `after_topic`)
3. Добавить поле `order_index int` — порядок показа, если на один момент несколько вопросов
4. В `useTestFlow.ts` — выбирать вопросы по `timing` + `topic_id` вместо случайного выбора
5. На экране `MotivationQuestionPage.tsx` — если `timing` связан с темой, показывать название темы
6. В админке `MotivationPage.tsx` — добавить выбор `timing` и привязку к теме

**В `motivation_answers`:** уже есть `asked_before_topic_id`. Нужно добавить поле для `timing`, чтобы различать "перед
темой" и "после темы".

## 10. Выбор заданий и порядка для линейного теста

**Что сейчас:** в линейном режиме темы перемешиваются случайно (`shuffleArray`), а внутри каждой темы задания выбираются
MST-алгоритмом (или случайно). Админ не контролирует ни порядок тем, ни выбор заданий.

**Что нужно:** Соня полностью контролирует линейный тест:

- Какие темы в каком порядке
- Какие конкретные задания в каждой теме и в каком порядке

**Что делать:**

1. Создать таблицу `linear_test_config`:
   ```sql
   linear_test_config (
     id uuid primary key,
     topic_id uuid references topics,
     item_id uuid references items,
     topic_order int,   -- порядок тем
     item_order int     -- порядок заданий внутри темы
   )
   ```
2. В админке — новый раздел "Линейный тест":
    - Список тем с возможностью задать порядок
    - Для каждой темы: список всех заданий, чекбоксы для включения + поле порядка
3. В `useTestFlow.ts` для `nonadaptive` режима:
    - Загружать темы и задания из `linear_test_config` (вместо shuffle + MST)
    - Порядок тем — по `topic_order`
    - Задания внутри темы — по `item_order`
4. Если `linear_test_config` пуста — fallback на текущее поведение (чтобы ничего не сломать)
